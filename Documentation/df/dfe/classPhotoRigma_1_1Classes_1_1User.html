Подробное описание

Класс по работе с пользователями и хранению данных о текущем пользователе.

Класс содержит методы для работы с данными пользователей, включая загрузку, обработку и сохранение данных. Идентификация пользователя происходит по данным из глобального массива $_SESSION. Также класс содержит ряд "заглушек" для будущей реализации методов по работе с пользователями и группами (добавление, изменение, удаление, вход). Этот класс реализует интерфейс User_Interface, но все методы пока являются заглушками.

Интерфейс для работы с пользователями и группами.

См. также

PhotoRigma\Include\log_in_file() Функция для логирования ошибок.
Необходимо сделать
Внедрить ряд методов по работе с группами и пользователями (добавление, изменение, удаление, вход).
Заметки
Используются константы с именами таблиц в базе данных.
Предупреждения
Класс зависит от глобального массива $_SESSION.

$user = new \PhotoRigma\Classes\User($db, $_SESSION);

private

Метод загружает права первого пользователя и первой группы из базы данных, извлекает имена полей (ключи) из JSON-поля user_rights и сохраняет их в свойство $user_right_fields.

Метод выполняет следующие шаги:

  4. Если данные отсутствуют или некорректны, сохраняется пустой массив. Этот метод является приватным и предназначен только для использования внутри класса.

◆ load_authenticated_user()

load_authenticated_user ( int $user_id)
private

Метод загружает данные аутентифицированного пользователя из базы данных, включая права пользователя и группы, обновляет дату последней активности и сохраняет данные в свойство $user.

Метод выполняет следующие шаги:

  1. Загружает данные пользователя из таблицы TBL_USERS по user_id.
  2. Если данные пользователя отсутствуют, сбрасывает сессию до гостя и вызывает метод load_guest_user().
  3. Обрабатывает права пользователя (user_rights) с помощью метода process_user_rights() и сохраняет их в свойство $user.
  4. Загружает данные группы пользователя из таблицы TBL_GROUP по group_id.
  5. Если данные группы отсутствуют, сбрасывает сессию до гостя и вызывает метод load_guest_user().
  6. Обрабатывает права группы (user_rights) с помощью метода process_user_rights() и объединяет их с данными пользователя через метод merge_user_with_group().
  7. Обновляет дату последней активности пользователя в таблице TBL_USERS. Этот метод является приватным и предназначен только для использования внутри класса.

◆ merge_user_with_group()

merge_user_with_group ( array $group_data)
private

Метод объединяет данные пользователя с данными его группы, применяя логику перезаписи значений.

Метод проходит по массиву данных группы ($group_data) и объединяет их с данными пользователя ($user) согласно следующей логике:

  1. Ключ name: Значение сохраняется в ‘$user['group’], а текущее значение$user['group']перемещается в$user['group_id'].
  2. Ключid: Пропускается.
  3. Для всех остальных ключей:
    • Если ключ существует в
    $user:
    • Если оба значения равны
    0, сохраняется0.
    • Иначе сохраняется
    1.
    • Если ключ отсутствует в
    $user, значение берётся из$group_data`. Этот метод является приватным и предназначен только для использования внутри класса.
См. также
PhotoRigma\Classes\User\$user Свойство, содержащее данные текущего пользователя.
Аргументы
array$group_dataАссоциативный массив, содержащий данные группы пользователя. Ключи должны соответствовать полям в таблице групп.
Заметки
Ключ id в данных группы игнорируется.
Предупреждения
Метод предполагает, что структура данных группы совпадает с ожидаемой. Неправильная структура может привести к ошибкам.

◆ process_user_rights()

process_user_rights ( $user_rights)
private

Метод обрабатывает поле user_rights, преобразуя его из строки JSON в массив прав.

Метод выполняет следующие шаги:

  1. Проверяет, существует ли поле user_rights. Если поле отсутствует, возвращается пустой массив.
  2. Проверяет, что поле является строкой. Если тип данных некорректен, выбрасывается исключение.
  3. Декодирует строку JSON в ассоциативный массив. Если декодирование завершается с ошибкой, выбрасывается исключение.
  4. Возвращает массив прав, готовый для добавления в свойство $user. Этот метод является приватным и предназначен только для использования внутри класса.
Аргументы
mixed$user_rightsЗначение поля user_rights. Может быть строкой JSON, пустым значением или null. Если значение не является строкой или содержит невалидный JSON, выбрасывается исключение.
Возвращает
array Ассоциативный массив прав пользователя. Если поле user_rights отсутствует или содержит некорректные данные, возвращается пустой массив.
Исключения
InvalidArgumentExceptionЕсли поле user_rights не является строкой.
InvalidArgumentExceptionЕсли поле user_rights содержит невалидный JSON.
Заметки
Метод возвращает пустой массив, если поле user_rights отсутствует или содержит некорректные данные.
Предупреждения
Поле user_rights должно содержать валидный JSON. Невалидные данные могут привести к исключению.

Пример вызова метода внутри класса:

$rights = $this->process_user_rights('{"edit": 1, "delete": 0}');
